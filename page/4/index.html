HMap.js

开源地图类库

起步 GITHUB

开发初衷

为了解决项目中类库版本不统一，方法重复和代码兼容性问题，还对一些GIS常用业务模型进行封装，解决快速开发问题。

优点

和原生openlaye相比，减少代码量，业务逻辑清晰

缺点

第一：可定制性和原生相比略微复杂；

第二：项目处于初期，暴露的bug可能比较多，文档相关不完善（正在积极解决问题和完善文档）
